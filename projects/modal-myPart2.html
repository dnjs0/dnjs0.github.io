연관 DB 테이블 소개

/* Schedule 테이블 */
CREATE TABLE schedule (
    schedule_id NUMBER PRIMARY KEY,
    user_id VARCHAR2(20) NOT NULL,
    title VARCHAR2(100) NOT NULL,
    content VARCHAR2(1000),
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    alarm_yn CHAR(1) DEFAULT 'N',
    alarm_time NUMBER,
    color VARCHAR2(20) DEFAULT '#4a6fdc',
    reg_date DATE DEFAULT SYSDATE,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);

/* User_Setting 테이블 */
CREATE TABLE user_setting (
    user_id VARCHAR2(20) PRIMARY KEY,
    default_view VARCHAR2(10) DEFAULT 'month',
    start_day_of_week NUMBER(1) DEFAULT 0,
    default_alarm_time NUMBER DEFAULT 30,
    theme VARCHAR2(10) DEFAULT 'light',
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);

문제 상황 및 해결 과정

문제 상황

일정이 많아질수록 캘린더 렌더링 성능이 저하되는 문제가 발생했으며, 특히 모바일 환경에서 사용자 경험이 크게 저하되었습니다.

해결 방법

가상 스크롤링(Virtual Scrolling) 기법을 도입하여 화면에 보이는 일정만 렌더링하도록 최적화했습니다. 또한 일정 데이터를 페이지 단위로 로드하는 무한 스크롤 방식을 구현하여 초기 로딩 시간을 단축했습니다.
