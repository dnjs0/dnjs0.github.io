담당 역할 및 주요 기능

  • 댓글 작성 및 관리 기능

연관 DB 테이블 소개

/* Member 테이블 */
CREATE TABLE member (
    member_id VARCHAR2(20) PRIMARY KEY,
    password VARCHAR2(100) NOT NULL,
    name VARCHAR2(50) NOT NULL,
    email VARCHAR2(100) UNIQUE NOT NULL,
    role VARCHAR2(20) DEFAULT 'ROLE_USER',
    reg_date DATE DEFAULT SYSDATE,
    update_date DATE DEFAULT SYSDATE
);

/* Comment 테이블 */
CREATE TABLE comment (
    comment_id NUMBER PRIMARY KEY,
    board_id NUMBER NOT NULL,
    member_id VARCHAR2(20) NOT NULL,
    content VARCHAR2(1000) NOT NULL,
    reg_date DATE DEFAULT SYSDATE,
    FOREIGN KEY (board_id) REFERENCES board(board_id),
    FOREIGN KEY (member_id) REFERENCES member(member_id)
);

문제 상황 및 해결 과정

문제 상황

기존 세션 기반 인증은 서버 확장성에 제한이 있었고, 보안 취약점이 발견되었습니다. 특히 CSRF 공격에 취약하고 세션 관리가 복잡했습니다.

해결 방법

Spring Security와 JWT를 도입하여 상태를 저장하지 않는(Stateless) 인증 방식으로 전환했습니다. 토큰 기반 인증으로 서버 확장성을 높이고, 보안성을 강화했습니다.

AccessToken과 RefreshToken을 분리하여 토큰 탈취 시 피해를 최소화했습니다.
